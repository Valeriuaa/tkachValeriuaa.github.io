1. Линейный алгоритм — треугольник

Условие: в равнобедренном треугольнике основание на 5 см меньше боковой стороны. Периметр = 35 см. Найти длины сторон.

Код (Java)

public class LinearTriangle {
    public static void main(String[] args) {
        double P = 35; // Периметр
        double d = 5;  // Разница между боковой и основанием

        double a = (P + d) / 3; // боковая
        double b = a - d;       // основание

        System.out.println("Боковая сторона a = " + a + " см");
        System.out.println("Основание b = " + b + " см");
    }
}

Проект АИС предназначен для централизованного учета сотрудников, департаментов, должностей и отпусков. В отчёте реализованы ER-диаграмма, диаграмма классов, диаграммы последовательности и состояний, а также прототип настольного приложения на Java Swing и модуль сохранения данных.

Система сохраняет данные в локальный сериализованный файл (data.ser) и поддерживает экспорт сотрудников, департаментов и должностей в текстовые файлы.

8. Техническое задание (фрагмент)

Ниже — выдержки из ТЗ на АИС «Отдел кадров»: назначение, функции, требования к платформе и безопасности.

Автоматизировать учёт сотрудников предприятия, обеспечить хранение и редактирование данных, разграничение прав доступа и экспорт/резервное копирование.

CRUD-операции по сотрудникам, департаментам и должностям; аутентификация; экспорт данных; логирование операций; восстановление данных при запуске.

Проектирование → Реализация → Тестирование → Внедрение и обучение пользователей.

9. Разработка мобильного приложения

Мобильное приложение «Отдел кадров» реализовано на Kotlin с архитектурой MVVM, локальным хранением в SharedPreferences (JSON) и хранением фотографий в приватной папке приложения.

Архитектура

MVVM: Repository (EmployeePrefs) → ViewModel → UI (Activity/Fragment). Используется RecyclerView + ListAdapter + DiffUtil для эффективного обновления списка.

Фрагменты кода (Kotlin)

// Пример: сохранение списка сотрудников в SharedPreferences (JSON)
fun saveEmployees(list: List) {
    val json = gson.toJson(list)
    prefs.edit().putString(KEY_EMPLOYEES, json).apply()
}

UI / UX

Главный экран — список карточек сотрудников с фото, имя/должность/отдел. Экран создания/редактирования с валидацией полей. Экспорт отчётов в PDF/Excel (в перспективе).

Тестирование

Покрытие тестами: операции репозитория и логика ViewModel. UI-тесты: проверки добавления/редактирования сотрудников, сохранения фото и восстановления после поворота экрана.

10. Руководство пользователя (краткое)

Короткое руководство по работе с АИС «Отдел кадров» и мобильным приложением: вход, добавление сотрудника, редактирование, экспорт и восстановление данных.

Вход в систему

Запустите приложение, введите логин/пароль. Для первой инициализации предусмотрены учётные записи admin/admin123 и hr/hr123.

Добавление сотрудника

Вкладка «Сотрудники» → «Добавить» → заполните поля: имя, фамилия, департамент, должность → ОК. Данные сохраняются автоматически.

Редактирование и удаление

Выберите запись в таблице → «Редактировать» или «Удалить». Права удаления ограничены для роли HR.

Экспорт и резервное копирование

Кнопка «Сохранить в файл» сохраняет список сотрудников в текстовый файл; регулярное копирование data.ser рекомендуется для резервирования.
